# UaAniSub — Telegram-бот продажу доступу до аніме-каналів

Бот працює через **Monobank Personal API** (особистий токен + банка).
Один платіж = довічний доступ. Платіж ідентифікується унікальним кодом
у полі «коментар» переказу.

## Як це працює

```
Юзер → /start → каталог → «Купити»
   → бот видає унікальний код (UAS-XXXXXX) і посилання на банку
   → юзер переказує суму, вписуючи код у коментар
   → Monobank шле StatementItem-webhook у бот
   → бот мэтчить код з коментаря → створює invite-link → шле юзеру
```

## Що потрібно

- Особистий токен Monobank: https://api.monobank.ua/ → «Отримати токен»
- **Банка (jar)** у додатку Monobank: створи, потім «Поділитись» → скопіюй
  частину URL після `/jar/` (це `sendId`, виглядає як `9xJa3M5Lp1`)
- Публічний HTTPS-URL для webhook (VPS / Cloudflare Tunnel / ngrok)

## Швидкий старт

1. **Залежності**
   ```
   pip install -r requirements.txt
   ```

2. **Налаштування**
   - Скопіюй `.env.example` → `.env` і заповни:
     - `BOT_TOKEN` — від [@BotFather](https://t.me/BotFather)
     - `ADMIN_IDS` — твій Telegram-ID через кому
     - `MONO_TOKEN` — особистий токен
     - `MONO_JAR_SEND_ID` — sendId банки (з URL `send.monobank.ua/jar/<тут>`)
     - `MONO_WEBHOOK_SECRET` — згенеруй довгу випадкову строку:
       ```
       python -c "import secrets; print(secrets.token_urlsafe(32))"
       ```
     - `WEBHOOK_BASE_URL` — публічний HTTPS-URL (див. нижче)

3. **Публічний URL для webhook**
   - **VPS з доменом:** `WEBHOOK_BASE_URL=https://yourdomain.com`
     (за reverse-proxy Caddy/nginx → `WEBHOOK_PORT`)
   - **Локально через тунель:**
     ```
     cloudflared tunnel --url http://localhost:8080
     ```
     Скопіюй виданий `https://*.trycloudflare.com` у `WEBHOOK_BASE_URL`.

4. **Запуск**
   ```
   python bot.py
   ```
   У логах має з'явитись:
   - «База даних ініціалізована»
   - «Monobank: знайдена банка ..., sendId=..., account=...»
   - «Webhook слухає 0.0.0.0:8080, шлях /monobank/webhook/...»
   - «Webhook зареєстровано в Monobank: https://...»
   - «Бот запущено!»

   Якщо в логах warning «Не вдалося зареєструвати webhook» — або URL
   вже встановлено раніше (це ОК), або URL недоступний ззовні.
   Personal API має rate limit 60 сек на цей запит.

## Підключення каналу

1. Створи приватний канал у Telegram.
2. Додай бота адміністратором з правом **Запрошувальні посилання** (Invite Users via Link).
3. У ЛС бота прийде повідомлення «Канал підключено» з `chat_id`.
4. Встанови ціну: `/setprice <chat_id> 99` (99 ₴).
5. (Опціонально) опис: `/setdesc <chat_id> Серії 1-12, озвучка UaAniSub`.

## Команди адміна

- `/channels` — список каналів-товарів
- `/setprice <chat_id> <UAH>` — ціна (десяткова, напр. `49.50`)
- `/setdesc <chat_id> <текст>` — короткий опис у каталозі
- `/toggle <chat_id>` — приховати/показати в каталозі
- `/remove <chat_id>` — видалити з каталогу (історія платежів залишається)
- `/stats` — продажі та виручка по каналах

## Edge cases

- **Юзер не вписав код у коментарі** → admin отримує алерт «Незіставлений платіж»,
  розбирається вручну.
- **Юзер вписав код невірно** → те саме.
- **Юзер вписав код як частину тексту** («UAS-XXXXXX доступ») → бот витягне токен.
- **Недоплата** → юзер отримує повідомлення «недостатня сума», admin — алерт.
- **Переплата** → юзер отримує доступ + «дякуємо за донат».
- **Прострочений код (>24 год)** → юзер може зробити нову покупку через /start.
- **Дубль webhook** (та сама транзакція) → ігнорується по `statement_id`.

## Обмеження Personal API (важливо)

- Mono **не підписує** webhook персонального API — єдиний захист це
  секретний URL (`MONO_WEBHOOK_SECRET` у шляху).
- Не можна автоматично повернути кошти — лише вручну з додатку.
- Rate limits: `client-info` і `set_webhook` — 1 раз / 60 сек.
- Якщо хочеш мати кілька банок (по каналах) — це потребує доробки;
  поточна версія працює з однією банкою.
